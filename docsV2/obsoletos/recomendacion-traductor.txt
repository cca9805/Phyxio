# 5. Regla de oro para Phyxio

internamente Phyxio debería depender de algo como:


### Español

```txt
/fisica-clasica/mecanica/cinematica/movimiento-rectilineo/mru
```

### Inglés

```txt
/classical-physics/mechanics/kinematics/rectilinear-motion/uniform-linear-motion
```

Así puedes cambiar la piel sin trasplantar el esqueleto.

## 6.3 `formulas.yaml`

Las fórmulas son prácticamente universales. Lo que cambia es la explicación.

### Propuesta

```yaml
- id: mru_position
  latex: x = x_0 + v t
  target: x
  variables:
    - x
    - x_0
    - v
    - t
  title: Ecuación de posición
  explanation: Permite calcular la posición en función del tiempo en MRU.
  rearrangements:
    - target: x
      latex: x = x_0 + v t
    - target: v
      latex: v = \frac{x - x_0}{t}
```

En inglés:

```yaml
- id: mru_position
  latex: x = x_0 + v t
  target: x
  variables:
    - x
    - x_0
    - v
    - t
  title: Position equation
  explanation: Computes position as a function of time in uniform linear motion.
  rearrangements:
    - target: x
      latex: x = x_0 + v t
    - target: v
      latex: v = \frac{x - x_0}{t}
```

### Lo que no cambia

* `id`
* `latex`
* `variables`
* `rearrangements`

### Lo que sí cambia

* `title`
* `explanation`

### Crear contexto

```javascript
import { createContext, useContext, useMemo, useState } from "react";
import es from "./ui/es";
import en from "./ui/en";

const I18nContext = createContext(null);

export function I18nProvider({ children }) {
  const [lang, setLang] = useState("es");

  const dict = useMemo(() => (lang === "en" ? en : es), [lang]);

  return (
    <I18nContext.Provider value={{ lang, setLang, dict }}>
      {children}
    </I18nContext.Provider>
  );
}

export function useI18n() {
  const ctx = useContext(I18nContext);
  if (!ctx) throw new Error("useI18n must be used inside I18nProvider");
  return ctx;
}
```

## Fase 3. Crear loader por idioma

Necesitas una función centralizada que resuelva contenido por idioma.

### Ejemplo conceptual

```javascript
export async function loadLeafFile({ lang, path, file }) {
  const mod = await import(`../data_v2_i18n/${lang}/${path}/${file}`);
  return mod.default ?? mod;
}
```

Pero como los imports dinámicos de Vite son quisquillosos, te recomiendo usar `import.meta.glob`.

### Mejor enfoque

```javascript
const mdModules = import.meta.glob("../data_v2_i18n/**/*.{md,yaml,yml,js,jsx}", {
  eager: false,
  query: "?raw",
  import: "default",
});

export async function loadRawContent(lang, relativePath) {
  const fullPath = `../data_v2_i18n/${lang}/${relativePath}`;
  const loader = mdModules[fullPath];
  if (!loader) throw new Error(`Content not found: ${fullPath}`);
  return await loader();
}
```

Ese enfoque te va mejor con Phyxio porque ya manejas mucho contenido file-based.
