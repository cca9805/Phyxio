# Common Errors: Types of Levers

## Conceptual errors

### Error 1: classifying by mechanical advantage

**Why it seems correct**
Mechanical advantage appears to summarize lever usefulness.

**Why it is incorrect**
Class depends on fulcrum-effort-load order, not on the value of [[VM]].

**Detection sign**
Every lever with [[VM]] greater than 1 is called second class.

**Conceptual correction**
First identify which element is in the middle, then interpret [[VM]].

**Mini-example**
A first-class lever may have [[VM]] greater than 1 if the fulcrum is close to the load.

### Error 2: thinking third class means bad design

**Why it seems correct**
Many third-class levers have force disadvantage.

**Why it is incorrect**
They may gain displacement, speed, or control, which are also mechanical goals.

**Detection sign**
Tweezers or the forearm are rejected because [[P]] may be larger than [[R]].

**Conceptual correction**
Relate class to intended function, not only to force.

**Mini-example**
The forearm sacrifices force to move the hand quickly and precisely.

## Model errors

### Error 3: measuring arms from the bar end

**Why it seems correct**
Total length is easy to see.

**Why it is incorrect**
[[d_P]] and [[d_R]] are measured from the fulcrum to the corresponding line of action.

**Detection sign**
The same length is used for both arms even though the fulcrum is not centered.

**Conceptual correction**
Draw the fulcrum and measure each arm from it.

**Mini-example**
In a wheelbarrow, the load arm is measured from the wheel to the load, not from the hands.

### Error 4: changing class when force changes

**Why it seems correct**
Changing [[P]] changes system behavior.

**Why it is incorrect**
Class changes only if the spatial order of fulcrum, effort, and load changes.

**Detection sign**
A lever is said to change from third to second class because load increases.

**Conceptual correction**
Keep configuration and force values separate.

**Mini-example**
A forearm remains third class even when lifting a heavier load.

## Mathematical errors

### Error 5: reversing the arms in advantage

**Why it seems correct**
Both arms appear as a ratio and are easy to swap.

**Why it is incorrect**
In this leaf, [[VM]] is read as [[d_P]] divided by [[d_R]].

**Detection sign**
A wheelbarrow with a longer effort arm gives [[VM]] smaller than 1.

**Conceptual correction**
First state which arm belongs to effort and which to load.

**Mini-example**
If [[d_P]] = 1.2 m and [[d_R]] = 0.4 m, [[VM]] = 3.

### Error 6: solving force without preserving torques

**Why it seems correct**
It may seem enough to divide forces directly.

**Why it is incorrect**
The ideal lever balances torques, not isolated forces.

**Detection sign**
The result gives [[P]] = [[R]] even though the arms are different.

**Conceptual correction**
Use equality between [[P]] times [[d_P]] and [[R]] times [[d_R]].

**Mini-example**
A 300 N load at 0.4 m balances with 100 N at 1.2 m.

## Interpretation errors

### Error 7: believing first class always multiplies force

**Why it seems correct**
Many first-class tools seem to make effort easier.

**Why it is incorrect**
First class only means fulcrum in the middle; advantage depends on arms.

**Detection sign**
[[d_P]] and [[d_R]] are not calculated because advantage is assumed automatic.

**Conceptual correction**
Classify first and compare arms afterward.

**Mini-example**
A seesaw with centered fulcrum has [[VM]] close to 1.

### Error 8: confusing effort with electrical power

**Why it seems correct**
The word effort/power has other meanings in physics.

**Why it is incorrect**
In levers, [[P]] means input force, measured in newtons.

**Detection sign**
[[P]] is expressed in watts.

**Conceptual correction**
Read [[P]] as applied force, not energy per time.

**Mini-example**
An effort of 100 N is a force, not 100 W.

## Quick self-control rule

Before finishing, identify fulcrum, effort, and load; decide which element is in the middle; measure arms from the fulcrum; use torques for calculation; interpret [[VM]] without confusing it with [[clase_palanca]].
