# Ejemplo tipo examen

## Enunciado

Se presenta un sistema clasico regular en el lenguaje de Pequenas oscilaciones y modos normales. Decide si puede usarse el formalismo avanzado, identifica la magnitud dominante [[omega]] y declara que estructura fisica se esta poniendo a prueba. Las magnitudes relevantes son [[eta]], [[M]], [[K]], [[omega]], [[A]], [[phi]], [[t]].

## Datos

- El sistema pertenece a un unico modelo analitico coherente.
- Las variables son suaves y estan dentro del dominio de validez.
- Las relaciones pedidas son la relacion definitoria, el criterio estructural y la relacion de caso limite del leaf.
- La respuesta debe incluir interpretacion, no solo manipulacion simbolica.

## Definicion del sistema

El sistema no se define solo por un dibujo. Se define por las variables que hacen significativo el criterio avanzado. En este ejemplo, la tarea es decidir si la descripcion formal preserva, separa, transforma o linealiza la mecanica original sin cambiar la pregunta fisica.

## Justificacion del modelo

El modelo es valido porque las variables se suponen regulares, el dominio es clasico y el objeto formal esta unido a un criterio fisico. No es valido solo porque la notacion este disponible. Si no se recupera el limite de particula libre, oscilador armonico, hamiltoniano autonomo o equilibrio estable, el modelo debe rechazarse.

## Resolucion simbolica

Usa las formulas del leaf en orden. Primero identifica [[omega]], despues aplica la relacion definitoria y luego el criterio de evolucion, transformacion, separacion o modo. La solucion simbolica significa que la estructura puesta a prueba es fisicamente coherente cuando se satisface la condicion requerida. Si la condicion falla, la formula indica el fallo en vez de ocultarlo.

## Sustitucion numerica

Elige una estimacion de escala unidad para el sistema original. Las formulas mostradas deben mantener [[omega]] en la escala esperada. Si una comprobacion estructural exacta deberia ser cero o uno, una deriva apreciable no es redondeo; indica una hipotesis rota. Si una frecuencia, accion o hamiltoniano transformado cambia mas de aproximadamente diez por ciento bajo una aproximacion pequena, el modelo debe revisarse.

## Validacion dimensional

Cada termino debe preservar la unidad declarada para [[omega]] o para su cantidad asociada. La comprobacion es fisica: una unidad incorrecta significa que la expresion formal ya no representa el mismo modelo.

## Interpretacion fisica

El resultado significa que el sistema es fisicamente coherente con la descripcion avanzada declarada. Indica que se preserva, que depende de las hipotesis del modelo y que fallaria si esas hipotesis se retiraran. Por tanto, la respuesta no es solo numerica; es una afirmacion sobre la estructura mecanica.

# Ejemplo de aplicacion real

## Contexto

Un profesor compara dos soluciones del mismo problema de mecanica avanzada. Una solucion es mas corta; la otra comprueba explicitamente la estructura de Pequenas oscilaciones y modos normales. La segunda se prefiere porque demuestra que la simplificacion no cambio el sistema fisico.

## Estimacion fisica

La estimacion usa la escala original de energia, accion, frecuencia o espacio de fases. El orden de magnitud debe permanecer razonable despues de aplicar la relacion avanzada. Un error de escala revela una variable equivocada, una aproximacion rota o una transformacion inadmisible.

## Interpretacion

Fisicamente, el ejemplo muestra que la mecanica avanzada es un lenguaje diagnostico. Ayuda a decidir si un resultado es valido, preservado, equivalente, separable o coherente. Por eso [[omega]] debe interpretarse, no solo calcularse.

En una correccion de examen, el profesor debe buscar tres evidencias: que el alumno justifica la linealizacion alrededor de equilibrio, que distingue frecuencia de forma modal y que explica la amplitud como participacion de un modo. Si solo calcula autovalores, falta la lectura mecanica de la vibracion colectiva.

## Comprobacion docente

Una respuesta completa debe contener una frase sobre validez, una frase sobre la magnitud dominante y una frase sobre el caso limite. Esto evita que el ejemplo se convierta en una sustitucion formal. El producto esperado del alumno es una decision fisicamente argumentada: el metodo avanzado se permite, se permite con condiciones o se rechaza porque han fallado las hipotesis del modelo.

Para completar el ejemplo, conviene pedir una verificacion verbal final: donde esta el equilibrio, que escala permite linealizar y que forma modal acompana a cada frecuencia. Esta verificacion evita que el alumno reduzca el problema a autovalores. Tambien permite interpretar si una excitacion real activa un modo peligroso, irrelevante o dominante.


## Extension didactica del ejemplo

El ejemplo tambien puede ampliarse pidiendo al alumno que excite el sistema con dos condiciones iniciales diferentes. Una puede activar casi un solo modo y otra puede mezclar varios. Esa comparacion muestra que los autovalores no son el final del problema: hace falta leer amplitudes, formas modales y energia repartida. La extension es util porque conecta el calculo matricial con una observacion fisica directa. El alumno aprende a decir por que un modo domina, por que otro apenas aparece y cuando la superposicion deja de ser una descripcion fiable.
