# Ejemplo tipo examen

## Enunciado

Se presenta un sistema clasico regular en el lenguaje de Transformaciones canonicas. Decide si puede usarse el formalismo avanzado, identifica la magnitud dominante [[C]] y declara que estructura fisica se esta poniendo a prueba. Las magnitudes relevantes son [[q]], [[p]], [[Q]], [[P]], [[K]], [[Fgen]], [[C]].

## Datos

- El sistema pertenece a un unico modelo analitico coherente.
- Las variables son suaves y estan dentro del dominio de validez.
- Las relaciones pedidas son la relacion definitoria, el criterio estructural y la relacion de caso limite del leaf.
- La respuesta debe incluir interpretacion, no solo manipulacion simbolica.

## Definicion del sistema

El sistema no se define solo por un dibujo. Se define por las variables que hacen significativo el criterio avanzado. En este ejemplo, la tarea es decidir si la descripcion formal preserva, separa, transforma o linealiza la mecanica original sin cambiar la pregunta fisica.

## Justificacion del modelo

El modelo es valido porque las variables se suponen regulares, el dominio es clasico y el objeto formal esta unido a un criterio fisico. No es valido solo porque la notacion este disponible. Si no se recupera el limite de particula libre, oscilador armonico, hamiltoniano autonomo o equilibrio estable, el modelo debe rechazarse.

## Resolucion simbolica

Usa las formulas del leaf en orden. Primero identifica [[C]], despues aplica la relacion definitoria y luego el criterio de evolucion, transformacion, separacion o modo. La solucion simbolica significa que la estructura puesta a prueba es fisicamente coherente cuando se satisface la condicion requerida. Si la condicion falla, la formula indica el fallo en vez de ocultarlo.

## Sustitucion numerica

Elige una estimacion de escala unidad para el sistema original. Las formulas mostradas deben mantener [[C]] en la escala esperada. Si una comprobacion estructural exacta deberia ser cero o uno, una deriva apreciable no es redondeo; indica una hipotesis rota. Si una frecuencia, accion o hamiltoniano transformado cambia mas de aproximadamente diez por ciento bajo una aproximacion pequena, el modelo debe revisarse.

## Validacion dimensional

Cada termino debe preservar la unidad declarada para [[C]] o para su cantidad asociada. La comprobacion es fisica: una unidad incorrecta significa que la expresion formal ya no representa el mismo modelo.

## Interpretacion fisica

El resultado significa que el sistema es fisicamente coherente con la descripcion avanzada declarada. Indica que se preserva, que depende de las hipotesis del modelo y que fallaria si esas hipotesis se retiraran. Por tanto, la respuesta no es solo numerica; es una afirmacion sobre la estructura mecanica.

# Ejemplo de aplicacion real

## Contexto

Un profesor compara dos soluciones del mismo problema de mecanica avanzada. Una solucion es mas corta; la otra comprueba explicitamente la estructura de Transformaciones canonicas. La segunda se prefiere porque demuestra que la simplificacion no cambio el sistema fisico.

## Estimacion fisica

La estimacion usa la escala original de energia, accion, frecuencia o espacio de fases. El orden de magnitud debe permanecer razonable despues de aplicar la relacion avanzada. Un error de escala revela una variable equivocada, una aproximacion rota o una transformacion inadmisible.

## Interpretacion

Fisicamente, el ejemplo muestra que la mecanica avanzada es un lenguaje diagnostico. Ayuda a decidir si un resultado es valido, preservado, equivalente, separable o coherente. Por eso [[C]] debe interpretarse, no solo calcularse.

En una correccion de examen, el profesor debe buscar tres evidencias: que el alumno no confunde cambio de notacion con transformacion canonica, que obtiene las variables nuevas desde la generatriz y que revisa la condicion de preservacion. Si el hamiltoniano cambia por dependencia temporal, esa correccion debe interpretarse como parte del modelo, no como error.

## Comprobacion docente

Una respuesta completa debe contener una frase sobre validez, una frase sobre la magnitud dominante y una frase sobre el caso limite. Esto evita que el ejemplo se convierta en una sustitucion formal. El producto esperado del alumno es una decision fisicamente argumentada: el metodo avanzado se permite, se permite con condiciones o se rechaza porque han fallado las hipotesis del modelo.

Para completar el ejemplo, conviene pedir una verificacion verbal final: que variables se abandonan, que variables se introducen y que condicion demuestra que el flujo conserva forma hamiltoniana. Esta verificacion evita que el alumno trate cualquier sustitucion como cambio canonico. Tambien permite comparar dos soluciones: la mejor es la que conserva reconstruccion e interpreta el hamiltoniano transformado.


## Extension didactica del ejemplo

El ejemplo tambien puede ampliarse pidiendo al alumno que proponga un cambio no canonico parecido al correcto. Al compararlos, se ve que dos sustituciones pueden simplificar la expresion y, sin embargo, solo una conserva la forma de las ecuaciones. Esta comparacion es muy eficaz para docencia avanzada porque muestra que el criterio canonico no es burocratico. Es la frontera entre describir el mismo sistema con mejores variables o construir sin querer otro sistema. La extension obliga a justificar reversibilidad, generatriz, corchetes y significado del hamiltoniano transformado.
