# Practica

## Nivel 1: reconocimiento

Asocia cantidades con escalas. Decide si `0.004 m`, `3000 m`, `2 x 10^-6 s` y `5 x 10^6 J` pertenecen a escala mili, kilo, micro o mega cuando corresponda.

Ejercicio: escribe el orden de magnitud aproximado de `7 m`, `80 m`, `900 m`, `0.02 s` y `0.0004 C`. No busques precision; busca escala.

## Nivel 2: aplicacion directa

Estima usando potencias de diez. Si una distancia es `600 m` y un tiempo `30 s`, aproxima como `10^3 m` y `10^1 s`, y estima la velocidad como `10^2 m/s`.

Ejercicio: estima la fuerza si una masa de `80 kg` acelera a `3 m/s^2`. Despues compara con el calculo fino y decide si el orden de magnitud coincide.

## Nivel 3: eleccion de herramienta

Decide cuando basta una estimacion y cuando hace falta calculo fino. Para detectar un error de unidades, el orden de magnitud suele bastar. Para comparar dos mediciones cercanas, no basta.

Ejercicio: indica si usarias estimacion o calculo detallado para calcular la altura de una mesa, calibrar un sensor, comprobar una velocidad absurda o comparar dos resistencias casi iguales.

## Nivel 4: interpretacion fisica

Convierte exponentes en lenguaje fisico. `10^-9 m` sugiere escala nanometrica; `10^3 m` escala kilometrica; `10^5 Pa` presion cercana a la atmosferica.

Ejercicio: interpreta `10^1 m/s`, `10^-3 kg`, `10^6 W` y `10^-6 C` diciendo que tipo de fenomeno podria tener esa escala.

## Nivel 5: error intencionado

Analiza esta estimacion incorrecta: una persona recorre `1000 m` en `100 s` y alguien estima `10^5 m/s`. Explica que operacion con exponentes se hizo mal y cual es la escala correcta.

Analiza otro caso: una longitud de `2 mm` se estima como `10^3 m`. Corrige el prefijo y explica por que el orden de magnitud ya revela el error antes de cualquier formula.

## Ruta de dominio

Nivel basico: debes transformar numeros en potencias de diez aproximadas y asociarlas con escalas familiares. El objetivo es detectar si algo pertenece a escala humana, microscopica, astronomica, lenta, rapida, debil o intensa.

Nivel intermedio: debes combinar ordenes de magnitud en formulas simples y usar el resultado para revisar si un calculo fino tiene sentido. Aqui no buscas precision decimal; buscas una alarma temprana ante unidades mal convertidas o resultados imposibles.

Nivel avanzado: debes justificar que terminos pueden despreciarse, comparar mecanismos dominantes y separar estimacion fisica de redondeo arbitrario. Una buena estimacion avanzada declara hipotesis, escala dominante y margen esperado de error.

## Practica de profundizacion

Nivel basico: inventa un ejemplo numerico sencillo relacionado con orden de magnitud, resuelvelo y escribe una frase con unidad y significado.

Nivel intermedio: cambia un dato por un factor `10` y predice el nuevo resultado antes de calcularlo. Luego verifica si la prediccion coincide.

Nivel avanzado: escribe un caso limite donde el procedimiento habitual pueda fallar. Explica que condicion fisica o matematica deja de cumplirse.
