# Common errors

## Error 1: choosing the wrong operation

It may seem reasonable to calculate a slope or an area whenever a graph appears, because those operations are common in physics. It fails when the question only asks for a value, an intercept, or a trend.

Detect it by asking what information the problem actually requests. If it asks "what temperature is reached", a coordinate is enough. If it asks "at what rate it changes", then slope becomes relevant. Correct it by separating visual reading from derived calculation.

## Error 2: losing the meaning of the variable

This error occurs when `x` and `y` are treated as empty letters instead of physical quantities. On a position-time graph, the horizontal axis is not a distance: it is time. Confusing them changes the physical interpretation completely.

Detect it by reading the axis labels aloud. Correct it by replacing each letter with its quantity: "position versus time", "energy versus height", or "amplitude versus frequency".

## Error 3: ignoring units

A coordinate without a unit may look clean, but it is not physically interpretable. Saying that velocity is `12` does not tell whether it is `12 m/s`, `12 km/h`, or a normalized scale.

Detect it by checking whether every final number includes the unit from the axis. Correct it by copying the exact graph unit and checking prefixes, powers of ten, or multiplicative scale factors.

## Error 4: confusing sign and magnitude

When a graph goes below zero, many students read only the distance from the axis and lose the sign. On a velocity-time graph, negative velocity does not mean "less speed"; it means motion opposite to the chosen positive direction.

Detect it by asking whether the sign represents direction, deficit, charge, relative displacement, or a value below a reference. Correct it by keeping the sign and explaining which reference defines zero.

## Error 5: applying a rule outside its domain

Extrapolating a trend beyond the data feels natural when the curve looks smooth. It fails because the model may change: a spring stops being linear, temperature levels off, or velocity becomes limited by drag.

Detect it by checking the actual interval of the graph and the experimental conditions. Correct it by limiting the conclusion to the displayed domain or explicitly justifying the model used for extrapolation.

## Quick detection rule

Before accepting a reading, state a complete sentence: "In the interval ..., the quantity ... with unit ... behaves ...". If you cannot complete interval, quantity, unit, and behavior, the reading is incomplete or you mixed operations.
