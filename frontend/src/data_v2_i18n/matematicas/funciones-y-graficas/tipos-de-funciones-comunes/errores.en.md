# Common errors

## Error 1: choosing the wrong operation

It may seem reasonable to fit a line to any data cloud because it is simple. It fails when curvature or scale shows another family. Detect it by inspecting residuals, concavity, and units. Correct it by comparing visual shape with physical meaning before calculating parameters.

## Error 2: losing the meaning of the variable

The same drawing may represent position, energy, field, or intensity. If you forget which quantity is on each axis, you interpret the function incorrectly. Detect it by asking what `x` represents and what `y` represents. Correct it by writing units next to every parameter.

## Error 3: ignoring units

A parameter without units may look correct and still be physically impossible. In `y = ax + b`, `a` does not have the same unit as `b`. Detect it by checking whether every term in the equation has the unit of `y`. Correct it with basic dimensional analysis.

## Error 4: confusing sign and magnitude

A negative slope, negative exponent, or negative phase is not automatically an error. It indicates decrease, decay, or displacement relative to a reference. Detect it by connecting the sign with the phenomenon. Correct it by explaining sense, not deleting the sign.

## Error 5: applying a rule outside its domain

A function may describe one interval well and fail outside it. A spring is not linear for every extension, and an inverse law may fail close to a real source. Detect it by checking assumptions and measured interval. Correct it by stating the validity domain.

## Quick detection rule

If you cannot say which physical parameter the chosen function represents, you have probably recognized only a superficial shape. Classification must end with unit, domain, and physical sentence.
